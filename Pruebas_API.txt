PRUEBAS DE API - Guia para Postman
====================================
Proyecto: Carritos-api
Prefijo base: http://localhost:8080/api/v1

INSTRUCCIONES PARA POSTMAN:
1. Abre Postman.
2. Crea una nueva coleccion llamada "Carritos API".
3. Para cada prueba, crea un nuevo request en la coleccion.
4. Los endpoints nuevos NO requieren autenticacion.
5. Los endpoints legacy (usuarios, solicitudes) SI requieren token Bearer.
   Para esos, primero ejecuta "5.1 Login", copia el token, y en la pestana
   "Authorization" de cada request legacy selecciona "Bearer Token" y pegalo.

NOTAS IMPORTANTES:
- Carritos se envian con snake_case: numero_carrito, estado_carrito, etc.
- Choferes se envian con estilo Go: Nombre, Licencia, Estado, etc.
- Horarios se envian con estilo Go: Turno, HoraInicio, HoraFin, etc.
- Despachos se envian con estilo Go: Fecha, NumeroCarrito, IDHorario, etc.


Levantar Backend en al terminal:
ejecutar en orden:

go build ./cmd/Carritos-api

go run ./cmd/Carritos-api

en Postman para generar token:
Haz un POST a /api/v1/login con cualquier email y password:
POST http://localhost:8080/api/v1/login
Content-Type: application/json
{"email": "test@test.com", "password": "pass"}
La respuesta será:
{"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}

Como usar el token
En Postman, para los requests a /usuarios o /solicitudes:
1. Ve a la pestaña Authorization
2. En el dropdown "Type", selecciona Bearer Token
3. Pega el token en el campo de la derecha
Eso hará que Postman agregue automáticamente el header:
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Nota importante
El token que devuelve el login es válido por 24 horas. Si expira, solo repites el paso del login para obtener uno nuevo.

8. PRUEBAS UNITARIAS CON TESTIFY
===============================================================================

Se crearon 5 archivos de prueba que usan la libreria github.com/stretchr/testify
con las siguientes funcionalidades:

  8.1 Testify Assert: aserciones con mensajes descriptivos (assert.Equal, assert.Len, assert.Empty, assert.ErrorIs)
  8.2 Testify Require: aserciones que detienen el test inmediatamente (require.NoError, require.Equal)
  8.3 Testify Mock: creacion de mocks con validacion de llamadas esperadas (mock.On, mock.AssertExpectations)

Archivos de prueba creados y sus casos:

  1. internal/storage/tests/sqlite_carritos_test.go (6 tests):
     - TestCarritoSQLite_CrearYObtener: crea y obtiene un carrito por numero
     - TestCarritoSQLite_GetCarritoByNumero_NoEncontrado: busca carrito inexistente
     - TestCarritoSQLite_ListarVacio: lista carritos en tabla vacia
     - TestCarritoSQLite_Actualizar: actualiza el estado de un carrito
     - TestCarritoSQLite_Eliminar: elimina (soft delete) y verifica que no se encuentre
     - TestCarritoSQLite_CrearDuplicatePK: inserta PK duplicada y espera error

  2. internal/storage/tests/sqlite_choferes_test.go (5 tests):
     - TestChoferSQLite_CrearYObtener: crea chofer y lo obtiene por ID
     - TestChofersQLite_GetChoferByLicencia_NoEncontrado: busca por licencia inexistente
     - TestChofersSQLite_ListarVacio: lista choferes en tabla vacia
     - TestChoferSQLite_Actualizar: actualiza el nombre de un chofer
     - TestChoferSQLite_Eliminar: elimina y verifica que no se encuentre

  3. internal/storage/tests/sqlite_horarios_test.go (5 tests):
     - TestHorarioSQLite_CrearYObtener: crea horario y lo obtiene por ID
     - TestHorarioSQLite_GetHorarioByID_NoEncontrado: busca horario inexistente por ID
     - TestHorarioSQLite_ListarVacio: lista horarios en tabla vacia
     - TestHorarioSQLite_Actualizar: actualiza la hora de fin de un horario
     - TestHorarioSQLite_Eliminar: elimina y verifica que no se encuentre

  4. internal/storage/tests/sqlite_despachos_test.go (6 tests):
     - TestDespachoSQLite_CrearYObtener: crea despacho y lo obtiene por ID
     - TestDespachoSQLite_GetDespachoByID_NoEncontrado: busca despacho inexistente
     - TestDespachoSQLite_ListarVacio: lista despachos en tabla vacia
     - TestDespachoSQLite_Actualizar: actualiza pasajeros actuales de un despacho
     - TestDespachoSQLite_ActualizarNoEncontrado: actualiza despacho inexistente
     - TestDespachoSQLite_Eliminar: elimina y verifica que no se encuentre

  5. internal/handlers/choferes_handler_test.go (5 tests - con testify/mock):
     - TestChoferHandler_Listar: mock GetAllChoferes, verifica lista en JSON
     - TestChoferHanler_Crear: mock CreateChofer, verifica HTTP 201 Created
     - TestChoferHandlerGetByID: mock GetChoferByID, verifica respuesta JSON
     - TestChoferHandlerGetByIDNoEncontrado: mock devuelve ErrNotFound, verifica HTTP 404
     - TestChoferHandlerCrearValidationError: envia campos invalidos, verifica HTTP 400

Como ejecutar las pruebas:

  1. Ejecutar todos los tests del proyecto:
     go test ./... -v

  2. Ejecutar tests de storage (unit tests con SQLite en memoria):
     go test ./internal/storage/tests/ -v

  3. Ejecutar tests de handlers (con testify/mock):
     go test ./internal/handlers/ -v

  4. Ejecutar un test en especifico por nombre:
     go test ./internal/storage/tests/ -run TestCarritoSQLite_CrearYObtener -v
     go test ./internal/handlers/ -run TestChoferHandler_Listar -v

  5. Ver cobertura de codigo:
     go test ./internal/storage/tests/ -cover
     go test ./internal/handlers/ -cover
